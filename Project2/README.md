## 1. Project Title and Problem

**Project Title:** AI vs. Human Text Detection

The goal of this project is to identify whether a submitted document was more likely written by a human or generated by artificial intelligence. This problem is important because AI-generated writing is becoming increasingly common in education, business, and professional communication. A simple AI/Human label is useful, but it does not fully explain why a document was classified that way.

This Project 2 application extends the original Project 1 machine learning detector by adding Large Language Model features. The final application does three things:

1. Classifies text as **Human-written** or **AI-written**.
2. Compares results across six traditional/deep learning classifiers.
3. Uses LLMs to explain the prediction and provide writing-coach feedback.

The application is deployed as a Streamlit app on Hugging Face Spaces.

---

## 2. Dataset Description

The project uses the original Project 1 labeled dataset for AI vs. Human text detection.

- **Human-written text:** label `0`
- **AI-generated text:** label `1`

The dataset contains text samples that were cleaned and transformed into machine-readable features. Preprocessing included text cleaning, lowercasing, punctuation removal, stopword handling, TF-IDF/vector-based representation, and linguistic feature extraction. The app also calculates document-level statistics such as word count, sentence count, vocabulary richness, average sentence length, and repeated terms.

The fine-tuning workflow also includes instruction-style examples that teach the LLM how to interpret ensemble model outputs, confidence scores, model disagreement, and document statistics.

## 3. Machine Learning / Deep Learning Models

The application keeps the original Project 1 model pipeline and includes all six required classifiers:

1. **Support Vector Machine (SVM)**
2. **Decision Tree**
3. **AdaBoost**
4. **Feedforward Neural Network style classifier (FNN)**
5. **LSTM-style text classifier**
6. **CNN-for-text style classifier**

The trained model files are stored in the `models/` directory:

```text
models/
├── svm.pkl
├── decision_tree.pkl
├── adaboost.pkl
├── fnn.pkl
├── lstm.pkl
└── cnn_for_text.pkl
```

The app uses lazy loading, which means models are loaded only when they are needed. This helps the Hugging Face Space run more reliably because all six models do not have to load at startup.

## 4. LLMs Used in the Application

Project 2 requires meaningful integration of at least two Large Language Models. This application includes two LLM-based tools layered on top of the six-model classifier system.

### LLM 1: AI Analyst

The AI Analyst receives:

- the final ensemble prediction,
- each model's individual prediction,
- each model's confidence score,
- the number of AI vs. Human votes,
- document statistics,
- feature-level evidence,
- and model interpretation notes.

It then produces a structured explanation of why the text was classified as AI-written or Human-written. This LLM does not replace the machine learning models; it explains the evidence produced by the models.

**Fine-tuned model repository:**

```text
halaszj/ai-text-analyst-flan-t5
```

### LLM 2: Writing Coach

The Writing Coach provides structured feedback to improve writing quality. It focuses on:

- sentence variation,
- clarity,
- specificity,
- natural voice,
- vocabulary variety,
- and reducing overly formulaic wording.

The Writing Coach helps move the project beyond simple classification by giving the user actionable writing guidance.

**Configured model repository:**

```text
halaszj/ai-writing-coach-flan-t5
```

If a fine-tuned model is not available or hosted inference fails, the app displays fallback guidance so the interface remains usable during a demo.

## 7. Results and What I Learned

The Project 1 machine learning models were able to classify text by learning patterns from labeled AI-written and human-written examples. The six-model comparison was useful because it showed whether the classifiers agreed or disagreed on a given document. The ensemble approach is stronger than relying on one model because it gives more context about confidence and uncertainty.

Project 2 showed that LLMs add value beyond prediction. The traditional machine learning models answer the question:

> Is this text likely AI-written or human-written?

The LLM tools answer the follow-up questions:

> Why did the models reach that conclusion?

and

> How can the writing be improved?

The most important lesson from implementation was that machine learning models and LLMs serve different roles. The classifiers are useful for prediction, while the LLMs are useful for explanation, interpretation, and communication. I also learned that deploying AI applications on Hugging Face Spaces requires careful dependency management, model loading design, and environment variable configuration.

2. Select **Streamlit** as the SDK.
3. Upload the runtime application files:

4. Add the model ID Variables:


7. Confirm the app shows the fine-tuned LLM status panel.
